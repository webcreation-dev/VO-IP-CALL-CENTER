TECHNICAL DETAILS
================================================================================

SIP CONFIGURATION FIELDS:
- Remote Host: IP:PORT or hostname:PORT
- Transport: UDP, TCP, TLS (default: UDP)
- Context: Dialplan context for incoming calls
- Authentication: Username/Password
- Registration Options: Interval, expiration, retries
- Outbound Proxy & Path Support

VALIDATION:
- Trunk name: 3-40 chars, alphanumeric with - and _
- Remote host: Valid IP:PORT format
- Credentials: 1-100 chars each
- Retry intervals: 10-3600 seconds
- Expiration: 60-7200 seconds
- Max retries: 1-50 attempts

ASTERISK INTEGRATION:
- AMI Commands:
  * pjsip show registrations (status)
  * module reload res_pjsip_wizard.conf (reload)
  * pjsip send register {id} (force registration)

- Registration Status Parsing:
  * Extracts: ID, server URI, auth, status, expiration
  * Normalizes: Registered, Rejected, Unregistered, Unknown

- File Format: pjsip_wizard.conf (ini-style)
  * [trunk-name] sections
  * Nested keys: endpoint/transport, outbound_auth/username
  * Auto-generated from database

SECURITY:
- JWT authentication (all endpoints)
- Role-based access control
- Tenant-based filtering
- Password stored in plaintext (security consideration)
- No response masking (security consideration)

FRONTEND INTEGRATION:
- TypeScript TrunksService (13 methods)
- Data transformation (snake_case ↔ camelCase)
- Status helpers (badges, formatting)
- Validation helpers
- 100% API endpoint coverage

================================================================================
WORKFLOW EXAMPLES
================================================================================

CREATE GLOBAL TRUNK:
1. POST /registrations with basic config
2. RegistrationsService.create() executes
3. SipTrunk entity created and saved to DB
4. ConfigFileService.generateFromDatabase() regenerates entire config
5. AMI reloads pjsip_wizard.conf
6. Trunk is active in Asterisk

ASSOCIATE AND ROUTE:
1. POST /registrations/:id/associate-tenant
2. Update tenant_id in DB
3. Regenerate and reload config
4. PATCH /registrations/:id/routing
5. Validate destination exists for tenant
6. Create dialplan extension via ExtensionsService
7. Incoming calls now route to destination

GET STATUS:
1. GET /registrations?with_status=true
2. Load all trunks from DB
3. For each: AmiService.getPJSIPRegistrations()
4. Match and parse status
5. Return combined trunk + status
